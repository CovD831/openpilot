# OpenPilot CLI 测试指南

本文档说明如何在 CLI 中测试 OpenPilot 的所有功能，包括 AGI 自主执行能力。

## 快速开始

### 1. 环境准备

```bash
# 激活 Conda 环境
conda activate openpilot

# 安装依赖（包含新增的 prompt-toolkit）
cd /mnt/c/Users/14235/Desktop/Projects/openPilot/Code
conda run -n openpilot python -m pip install -e .

# 配置 LLM API（必需）
cp .env.example .env
# 编辑 .env 文件，填入你的 API key
```

### 2. 启动交互式 CLI

```bash
openpilot run
```

### 3. 系统命令（以 / 开头）

在 CLI 中输入 `/` 后，会自动显示命令补全菜单：

```
/help        - 显示帮助
/config      - 检查配置
/plan        - 生成任务计划
/execute     - 执行完整工作流
/autopilot   - 🚀 AGI 自主模式
/task        - 任务管理
/report      - 报告生成
/memory      - 记忆系统状态
/clear       - 清屏
/exit, /quit - 退出
```

**使用方法**：
- 输入 `/` 后按 Tab 键查看所有命令
- 使用上下箭头键浏览命令历史
- 按 Ctrl+R 搜索历史命令
- 按 Enter 确认选择

**快捷键**：
- `↑` / `↓` - 浏览命令历史
- `Tab` - 命令补全
- `Ctrl+R` - 搜索历史命令
- `Ctrl+C` - 取消当前输入
- `Ctrl+D` 或 `exit` - 退出

## 🚀 AGI 自主执行能力测试

### 什么是 Autopilot 模式？

Autopilot 模式是 OpenPilot 的 **AGI 能力核心**，实现了：
- ✅ 完全自主的任务理解和分解
- ✅ 自动工具选择和编排
- ✅ 自动代码生成和执行
- ✅ 自动结果验证和优化
- ✅ 持续学习和策略改进

**与普通模式的区别**：
- 普通模式：每个步骤都需要用户确认
- Autopilot 模式：自动批准低风险和中风险操作，只在高风险时询问

## OP-16: 任务日志模块测试

### 列出所有任务

```bash
openpilot task list
```

**预期输出：**
- 显示所有有日志记录的任务
- 显示每个任务的日志条目数量

### 创建任务日志

```bash
# 创建任务
openpilot task log my_first_task created

# 更新任务状态
openpilot task log my_first_task status_changed --old-status planned --new-status in_progress

# 添加备注
openpilot task log my_first_task note_added --note "完成了初步设计"

# 记录阻塞（必须提供原因）
openpilot task log my_first_task blocked --blocked-reason "等待 API 密钥审批"

# 解除阻塞
openpilot task log my_first_task unblocked --note "API 密钥已获取"

# 完成任务
openpilot task log my_first_task completed
```

**预期输出：**
- 每次操作后显示 `✓ Logged [event_type] for task: [task_id]`

### 查看任务历史

```bash
# 查看完整历史
openpilot task history my_first_task

# 只查看状态变更
openpilot task history my_first_task --event-type status_changed

# 只查看阻塞事件
openpilot task history my_first_task --event-type blocked
```

**预期输出：**
- 表格形式显示任务历史
- 包含时间戳、事件类型、详细信息

### 可用的事件类型

- `created` - 任务创建
- `status_changed` - 状态变更（需要 `--old-status` 和 `--new-status`）
- `blocked` - 任务阻塞（需要 `--blocked-reason`）
- `unblocked` - 解除阻塞
- `note_added` - 添加备注（需要 `--note`）
- `completed` - 任务完成
- `skipped` - 任务跳过
- `timeline_updated` - 时间线更新

## OP-03/OP-04: 记忆模块与偏好复用测试

### 记忆系统架构

OpenPilot 实现了四层记忆系统：

1. **短期记忆**（Short-term）：当前会话上下文
2. **长期记忆**（Long-term）：用户偏好、工作习惯
3. **任务记忆**（Task）：历史任务执行记录
4. **技能记忆**（Skill）：可复用的流程模板

### 测试记忆功能

```bash
# 1. 正常运行（启用记忆）
openpilot run --once "研究 AI Agent 市场并生成报告"

# 2. 禁用记忆运行（对比测试）
openpilot run --once "研究 AI Agent 市场并生成报告" --ignore-memory

# 3. 查看记忆文件
ls -la data/memory/
cat data/memory/long_term.jsonl | jq .
cat data/memory/task.jsonl | jq .
```

**预期行为：**
- 首次运行时，系统会学习用户偏好
- 再次运行相似任务时，系统会复用历史偏好
- 使用 `--ignore-memory` 时，系统不会读取或应用历史记忆
- 记忆文件以 JSONL 格式存储在 `data/memory/` 目录

### 验证偏好复用

```bash
# 场景：多次运行相似任务，观察系统是否学习偏好

# 第一次：系统会询问或使用默认设置
openpilot run --once "生成项目周报"

# 第二次：系统应该复用之前的偏好（如报告格式、输出风格）
```

## OP-19: 自主判断与置信度控制测试

### 置信度驱动的自主度

系统会根据历史成功率和用户偏好计算置信度，决定是否需要询问用户：

- **低风险 + 高置信度** → 自动执行
- **中风险 + 高置信度** → 通知后执行
- **高风险** → 必须确认
- **禁止操作** → 阻断

### 测试自主判断

```bash

openpilot run --once "整理会议记录"

# 3. 查看自主度决策日志
cat logs/openpilot.jsonl | jq 'select(.event == "autonomy_decision_created")'
```

**预期行为：**
- 首次任务会有更多确认点
- 重复成功后，低风险操作会减少确认
- 高风险操作始终需要确认
- 日志记录每个决策的置信度和原因
