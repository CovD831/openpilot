## 文档维护规则

### 新增模块时

1. 在 `产品方案任务流程.md` 中标记任务为"已完成"
2. 创建 `OP-XX-完成总结.md` 文档
3. 更新 `API.md` 中的模块接口
4. 更新 `测试指南.md` 中的测试方法
5. 更新本 README 的进度部分

### 架构变更时

1. 更新 `OpenPilot-产品架构与路线图.md`
2. 如有必要，更新 `产品方案任务流程.md`
3. 通知所有团队成员

### 文档命名规范

- 架构文档：`OpenPilot-XXX.md`
- 实现文档：`产品方案XXX.md`
- 完成总结：`OP-XX-完成总结.md`
- 指南文档：`XXX指南.md`
